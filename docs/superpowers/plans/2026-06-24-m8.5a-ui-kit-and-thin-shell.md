# M8.5a — `@shadowcat/ui-kit` + Thin Shell Implementation Plan

> **For agentic workers:** Execute via `mainline-plan-execution` (this project's
> Fable-class execution skill — inline per-task spec-compliance check + one final
> branch review; NOT per-task subagent dispatch). Steps use checkbox (`- [ ]`)
> syntax for tracking.

**Goal:** Extract the Svelte-runtime seams that every UI module imports into a new
shared `@shadowcat/ui-kit` package, and rename the app package `@shadowcat/ui` →
`@shadowcat/shell`, establishing the package layering the rest of M8.5 builds on.

**Architecture:** `appContext`, `Surface`, the `i18n` Svelte adapter, and the
`sceneInteraction` bridge move out of the shell's `lib/` into `@shadowcat/ui-kit`
(depends on `core` + `render` types; consumed by the shell and, later, every
module). This breaks the would-be shell↔module import cycle. Pure refactor — zero
behavior change. This is slice **a** of three (a → b → c); b (entry package) and c
(per-element split) get their own plans after a merges.

**Tech Stack:** pnpm workspaces, Vite 8, Svelte 5 (runes), Vitest 4 + jsdom +
`@testing-library/svelte`, TypeScript 5.6.

## Global Constraints

- **No behavior change.** This is a mechanical extraction + rename. The only proof
  of correctness is: every existing test still passes, the client still builds to
  `dist/`, and the binary-served Playwright e2e stays green.
- **`@shadowcat/core` is Svelte-free** (ARCHITECTURE.md invariant #7). The moved
  files use the Svelte runtime, so they go in `ui-kit`, never `core`.
- **`dist/` output path is unchanged.** Vite `build.outDir` stays `../../../dist`;
  `rust-embed` validates `../../dist/` (from the server crate) at compile time, so
  CI must keep building the client before any `cargo` step.
- **Cross-platform.** Use `git mv` (portable); no OS-specific path assumptions.
- **Permissive licenses only.** No new runtime dependencies are added in this slice.
- **Refactor-shaped cycle.** For pure file moves there is no failing-test-first
  step; the cycle is *move → repoint → typecheck/test/build green → commit*. New
  code (the `index.ts` barrel) is verified by consumers' typecheck + the moved
  tests passing under the new package.
