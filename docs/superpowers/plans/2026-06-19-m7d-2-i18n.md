# M7d-2 — i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: this project executes plans with
> the **mainline-plan-execution** skill (inline, per-task spec-compliance check +
> a single final branch review) — NOT subagent-driven-development or
> executing-plans. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A framework-neutral i18n seam — a neutral `I18n` core in
`@shadowcat/core`, a Svelte `t` adapter, one `en` catalog, every UI string routed
through `t()`, and a locale switcher.

**Architecture:** The `I18n` primitive lives in `@shadowcat/core` (no framework
imports), mirroring `DocumentStore`/`ContributionRegistry` — `t`/`locale`/
`setLocale`/`subscribe`. The `ui` package wraps it with `createSubscriber` (the
same bridge `<Surface>` uses) into a reactive `t`, holds one `en` catalog, and
exposes `t` to in-world components via `AppContext.t` and to the pre-context entry
views via a direct import. A Vue host would adapt the same `I18n` object.

**Tech Stack:** TypeScript, Svelte 5, Vitest.

## Global Constraints

- The `I18n` core is **framework-neutral** (no Svelte/DOM imports), subscribe/
  snapshot like the other core primitives. Missing key → the key (visible, never
  throws). Minimal `{name}` interpolation; ICU/pluralization deferred.
- Every user-facing string in the entry views + `core-ui` panels routes through
  `t('key')`. The `en` catalog values must **exactly match the current literals**
  so the existing render tests (which assert on "Log in", "Username", "Your
  worlds", etc.) stay green.
- TDD for the testable units (core `I18n`, the Svelte reactivity adapter, then the

- [ ] **Step 3: Implement the core**

`src/client/core/src/i18n.ts`:

```ts
// Framework-neutral i18n primitive: subscribe/snapshot like DocumentStore and
// ContributionRegistry, so any framework (Svelte via createSubscriber, Vue, …)
// can read t() reactively. Minimal {name} interpolation; ICU/plural deferred.
export type Messages = Record<string, string>;
export type I18nParams = Record<string, string | number>;
export type Listener = () => void;

export class I18n {
  #locale: string;
  #catalogs: Record<string, Messages>;
  #listeners = new Set<Listener>();

  constructor(locale: string, catalogs: Record<string, Messages>) {
    this.#locale = locale;
    this.#catalogs = catalogs;
  }

  get locale(): string {
    return this.#locale;
  }

  get locales(): string[] {
    return Object.keys(this.#catalogs);
  }

  setLocale(locale: string): void {
    if (locale === this.#locale) return;
    this.#locale = locale;
    for (const fn of this.#listeners) fn();
  }

  /** Look up `key` in the current locale; missing key → the key itself. */
  t(key: string, params?: I18nParams): string {
    const msg = this.#catalogs[this.#locale]?.[key] ?? key;
    return params ? interpolate(msg, params) : msg;
  }

  subscribe(listener: Listener): () => void {
    this.#listeners.add(listener);
    return () => this.#listeners.delete(listener);
  }
}

/** Replace `{name}` with params; an unknown placeholder is left intact. */
function interpolate(msg: string, params: I18nParams): string {
  return msg.replace(/\{(\w+)\}/g, (_, k: string) =>
    k in params ? String(params[k]) : `{${k}}`,
  );
}
```

`src/client/ui/src/lib/i18n.svelte.ts`:

```ts
import { createSubscriber } from "svelte/reactivity";
import { I18n, type I18nParams } from "@shadowcat/core";
import { en } from "../locales/en";

/** The app's single i18n instance (one `en` catalog for now). */
export const i18n = new I18n("en", { en });

const subscribe = createSubscriber((update) => i18n.subscribe(update));

/** Reactive translate: reading it in a rune context re-runs on setLocale. */
export function t(key: string, params?: I18nParams): string {
  subscribe();
  return i18n.t(key, params);
}

export type TFunc = typeof t;
```
