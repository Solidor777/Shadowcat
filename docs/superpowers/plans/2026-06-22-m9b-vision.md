# M9b — Server Vision Raycasting + Live Visibility Mask: Plan

> **For agentic workers:** execute with **`mainline-plan-execution`** (inline enumerative
> per-task spec check + ONE dispatched final branch review; buddy-check it — this is
> security-sensitive: a vision bug is a secret leak). Checkbox steps. **Read the M9 spec §6

**Goal:** Server-authoritative per-player vision: a Rust raycaster computes each player's
visibility polygon against `blocksSight` walls, `geo`-unioned across their controlled tokens,
dispatched over a real **`vision`** `SceneDerived` channel; the client maps polygons →
`VisibilityInput` → an engine-owned **two-state** fog shader (visible vs not-visible). Persistent
"explored" fog + GM see-all/see-as-player = **M9c**.

**Spec:** [`../specs/2026-06-22-m9-walls-vision-fog-design.md`](../specs/2026-06-22-m9-walls-vision-fog-design.md)
§6 (vision), §10 (decisions). Builds on M9a walls + the M8a/M8c `SceneDerived` transport + the
M8c `Compositor` mask slot.

## Design decisions (resolved autonomously; flag for review)

These refine M9 §6, which fixed the architecture but left implementation details open. Decided on
the merits; **surface for the user** — vision is security-sensitive and the fog semantics are a
user-visible behavior.

1. **Vision bound (no scene dimensions yet, §7).** Rays that hit no wall need a far boundary.
   Use the **axis-aligned bounding box of the scene's `blocksSight` wall endpoints + the
   viewpoint, expanded by a margin** (e.g. 1 grid cell); add the 4 box edges to the raycast set so
   every ray terminates. No arbitrary sight radius (geometric vision only, §11 — range senses are
   Phase 2). If a scene has no walls, the box is just the viewpoint margin → a small fully-visible
   area (degenerate but safe; a wall-less scene has nothing to occlude anyway → prefer `mode:"all"`).
2. **Fog semantics — extend `VisibilityInput` with a `mode`.** The M8 "empty `visible` = identity"
   placeholder conflates "no fog" with "see nothing." Replace with:
   `VisibilityInput { mode: "all" | "masked"; visible: Polygon[]; explored?: Polygon[] }`.
   - `mode:"all"` → no fog (GM in M9b; the M8 identity case maps here).
   - `mode:"masked"` → fog covers everything **outside** `visible`; **empty `visible` = full fog**
     (a player with no controlled tokens sees nothing — the correct default, not "see all").
   Touches `compositor.ts`, `pixi-backend.setVisibility`, `engine.toVisibility`, and the M8c
   identity tests (update them to `mode:"all"`).
3. **Controlled tokens = `token.owner == ctx.user_id`** (the only ownership signal today; M10 adds
   actor-linking). A player's vision = `geo`-union of the visibility polygon from each owned token's
   center in the active scene. **GM → `mode:"all"`** (sees everything; see-as-player is M9c).
4. **Payload shape (D-V1, compact):** `{ mode, polygons: [[x0,y0,x1,y1,…], …] }` (flat point
   arrays, scene coords). The client rasterizes. Computed per-recipient in `compute_derived`.

## Global constraints
- **#1/#4 per-recipient:** a player only ever receives **their** visible polygon; a `gm_only` wall
  is never transmitted but **still occludes** (the server raycasts the full wall set). Vision is the
  first real per-recipient `compute_derived` consumer.
- **#3 vision exemption:** server-authoritative, NO client prediction. The mask arrives via
  `SceneDerived` **after** the move event it derives from (computed-at-seq watermark, already built).
- **#7 clean-room:** the visibility-polygon technique cites public computational-geometry literature
  only (angular sweep / "ray casting to endpoints"); `geo` for union. No proprietary VTT/engine source.
- **#10:** the fog shader is a 2D PixiJS filter/mask (the `realtime-rendering` skill is **N/A**).
- **No raw `console.log`/`println!`; commit per task; push at the M9b milestone end (CI green).**

### Task 1: `geo` dependency + the visibility-polygon raycaster (pure Rust)

**Files:** `src/server/Cargo.toml` (+`geo`), `src/server/src/scene/vision.rs` (new), `scene/mod.rs`
(mod decl); tests in `vision.rs`.
- Add `geo` (polygon union/types). Confirm the version + that `cargo build` resolves it.
- `visibility_polygon(viewpoint: (f64,f64), walls: &[Segment], bound: Rect) -> Vec<(f64,f64)>` —
  the clean-room angular sweep: collect wall + bound-box endpoints; per endpoint cast 3 rays
  (`θ`, `θ±ε`); nearest segment hit per ray; sort hits by angle → polygon vertices.
  *Source: standard 2D visibility-polygon / "ray casting to endpoints" (Red Blob Games / CG literature).*
- `union_polygons(polys: &[Vec<(f64,f64)>]) -> Vec<Vec<(f64,f64)>>` via `geo::union` (MultiPolygon).

  room (4 walls) → the viewpoint sees only inside; two tokens' polygons union to the combined area;
  a ray grazing a corner is handled (the ±ε rays). Headless, deterministic.

### Task 2: the `vision` channel in `compute_derived`

**Files:** `src/server/src/scene/mod.rs` (the `compute_derived` "vision" arm + a per-player compute
reading the ECS), `vision.rs`; tests in `mod.rs` + `tests/scene_derived.rs`.
- `compute_derived("vision", ecs, ctx)`:
  - GM (`ctx.world_role == Gm`) → `Some(json!({ "mode": "all" }))`.
  - else collect the player's controlled tokens (`doc_type=="token"`, `owner==ctx.user_id`) in the
    active scene; the scene's `blocksSight` wall segments; compute each token's `visibility_polygon`
    (bound = wall+token bbox + margin); `union_polygons`; return
    `Some(json!({ "mode": "masked", "polygons": [...] }))` (empty `polygons` if no tokens).
  - Per-recipient + #4: only this player's polygon is ever in the payload.
- The egress recompute (coalesced, computed-at-seq, fingerprint-deduped) is **already built** —
  vision is just a new channel arm. A token move or wall change re-runs it for affected players.
- [ ] Test: a scene with a wall + a player-owned token → `vision` payload has `mode:"masked"` +
  polygon(s); a GM → `mode:"all"`; a player with no tokens → `mode:"masked"` + empty `polygons`;
  moving the token changes the polygon (fingerprint differs → re-pushed). Mirror `scene_derived.rs`.

### Task 4: engine-owned two-state fog shader (Pixi)

**Files:** `src/client/render/src/pixi-backend.ts` (`setVisibility` draws fog), `backend.mock.ts`
(record the mode), `backend.ts`; a Playwright assertion.
- `setVisibility(input)`: `mode:"all"` → clear the fog (no occlusion). `mode:"masked"` → fill the
  fog-affected area with an opaque fog and **cut out** the `visible` polygons (a Pixi mask: draw fog
  over the viewport, set the polygons as an inverse mask / use a `Graphics` hole). Fog covers the
  `mask` layer slot over the fog-affected layers (tokens/drawings/background). Two-state only (visible
  vs not); `explored` dimming = M9c.
- The fog must cover the visible **viewport**, not just the polygon bbox (pan/zoom-stable). Reuse the
  camera transform; the fog Graphics lives in the `mask` layer (world-space) sized to the wall bbox +
  generous margin, OR a screen-space filter. **Decision:** world-space fog rect = the vision bound
  box (already the polygon's outer bound) — everything outside a token's vision is within the box and
  fogged; beyond the box there's nothing to occlude (no walls). Document this.
- Mock: `visibility` already records the input; assert `mode`. Pixi = GL (Playwright).
- [ ] Mock test (mode recorded) + keep render unit suite green; **Playwright:** a player sees fog
  occlude beyond a wall (needs a player session — see Task 6 note). Implement; pixi-invariant grep.
- [ ] **Commit:** `feat(m9b): engine-owned two-state fog shader over the mask slot`

## Buddy-check directive
M9b is **security-critical**: a raycaster error, a per-recipient leak (sending one player another's
polygon, or transmitting a `gm_only` wall), or a watermark slip is a secret reveal — the exact "a
peek can ruin the game" failure §5 calls load-bearing. **Run a buddy-check at the final branch
review**, focusing on: visibility-polygon correctness (corner ±ε rays, the bound box, collinear
walls), `geo`-union robustness, **per-recipient isolation** (each payload contains only that
recipient's polygon; full wall set occludes but `gm_only` walls aren't transmitted), the GM
`mode:"all"` path, the empty-polygons = full-fog (no "see all" leak for a token-less player), the
fog mask cut-out (no off-by-one revealing a sliver), and the computed-at-seq watermark (mask never
applied before its move event). Clean-room provenance (#7) checked per-file.
