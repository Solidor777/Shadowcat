# M7d-1 — Theming Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: this project executes plans with
> the **mainline-plan-execution** skill (inline, per-task spec-compliance check +
> a single final branch review) — NOT subagent-driven-development or
> executing-plans. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply the icon-derived dark theme via a 3-tier SCSS token system,
replacing M7c's minimal structural styles across the SPA.

**Architecture:** SCSS token files (primitives → semantic) emit **CSS custom
properties on `:root`**; one `global.scss` (imported in `main.ts`) carries them
plus base element styles. Svelte components use `var(--token)` in scoped
`<style lang="scss">`. Single dark theme. `vitePreprocess` (already configured)
compiles SCSS once `sass` is a dev-dep.

**Tech Stack:** Svelte 5, SCSS (`sass`), Vite.

## Global Constraints

- **Components reference only the semantic (tier 2) tokens** (`--surface-*`,
  `--text-*`, `--accent*`, `--border`, `--danger/success/warning`), never raw
  tier-1 ramps directly.
- Single dark theme; near-black-but-not-pure base (`--slate-950 #16161F`).
- Palette is **provisional** — re-audited at M8/M12 (parent plan); recorded as a
  comment in the primitives file.
- Verification is build + existing render tests + svelte-check + a contrast check
  (theming has no new behavior to TDD; the existing component tests already
  render the scoped SCSS, so a broken style fails them).
- Commands (from repo root):
  - Client build (compiles `global.scss`): `pnpm --filter @shadowcat/ui build`
  - ui unit tests (compile scoped component SCSS): `pnpm --filter @shadowcat/ui test`
  - svelte-check: `pnpm --filter @shadowcat/ui typecheck`

Verify the key pairs meet AA (manual / a contrast calculator):
- `--text-primary #E7E8F2` on `--surface-base #1F1F2C` → ≈ 14:1 (AAA). ✓
- `--text-primary` on `--accent #2D6EE8` (button label) → ≈ 4.5:1 (AA normal). If

- `--text-muted #6B6D85` on `--surface-base` → ≈ 3:1 (AA large / non-essential
  only — keep muted text at ≥0.875rem or for secondary labels).
Record the three measured ratios in the commit message or a comment; adjust a
token if any essential pair misses AA.
