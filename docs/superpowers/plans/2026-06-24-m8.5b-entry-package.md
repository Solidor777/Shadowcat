# M8.5b — Swappable `@shadowcat/module-entry` Implementation Plan

> **For agentic workers:** Execute via `mainline-plan-execution` (inline per-task
> spec-compliance check + one final branch review / buddy-check). Steps use
> checkbox (`- [ ]`) syntax.

**Goal:** Extract the pre-world entry flow (setup / login / world-select) out of
the shell into a swappable `@shadowcat/module-entry` package exposing a single
`<Entry>` component with an `{ onAuthenticated, onEnterWorld }` callback contract;
move entry's REST into a co-located `entryApi`. The shell keeps the
session/world lifecycle (identity, locale, last-world auto-enter).

**Architecture:** Slice **b** of M8.5 (a done, c next). `<Entry>` owns its internal
`setup → login → world-select` step state and its REST (`getConfig`, `login`,
`setup`, `listWorlds`, `createWorld`, plus an auth probe `getMe`). The shell's boot
resolves identity + last-world and either auto-enters (skipping entry) or renders
`<Entry>`. Pure refactor — **zero behavior change** (verified by the unchanged
binary-served Playwright e2e). Builds on the layering from M8.5a
(`@shadowcat/ui-kit`).

**Tech Stack:** pnpm workspaces, Vite 8, Svelte 5 (runes), Vitest 4 + jsdom +
`@testing-library/svelte`, TypeScript 5.6.

## Global Constraints

- **No behavior change.** Mechanical extraction + a boot-flow split that preserves
  the exact observable sequence (setup→login→world-select; authed-with-valid-
  last-world auto-enters; saved locale applied before world-select renders). The
  binary-served Playwright e2e (`src/client/shell/e2e/entry-flow.spec.ts`) is the
  proof and must stay green unedited.
- **`@shadowcat/core` stays Svelte-free** (invariant #7). Entry depends on
  `@shadowcat/ui-kit` (for `t`) + `@shadowcat/types`, not on the shell.
- **Entry must not import the shell** (the shell imports entry — the reverse would
  cycle). Entry communicates only via its `{ onAuthenticated, onEnterWorld }` props.
- **`dist/` path + CI ordering unchanged.** No new package builds to `dist/`; only
  the shell (`@shadowcat/shell`) does.
- **Stale-ref sweep is whole-tree.** After moving endpoints/views, grep the whole
  tree (no `--include` allowlist — `.rs`/`.sh`/`.toml` too) for any dangling
  reference to a moved symbol path.
- **Refactor-shaped cycle:** move/rewire → typecheck/test/build green → commit. New
  code (the `<Entry>` orchestrator) is test-driven where it carries real logic
  (step routing).

## File Structure

New package `src/modules/entry/` → `@shadowcat/module-entry` (workspace glob
`src/modules/*` already matches; remove the placeholder `src/modules/.gitkeep`):
- `package.json`, `tsconfig.json`, `svelte.config.js`, `vitest.config.ts`, `vitest.setup.ts`
- `src/index.ts` — exports `Entry`.
- `src/Entry.svelte` — the orchestrator (internal step state machine + contract).
- `src/Entry.test.ts` — step-routing tests (moved from `App.test.ts`).
- `src/entryApi.ts` — co-located REST (`getConfig`, `login`, `setup`, `listWorlds`,
  `createWorld`, `getMe` auth probe) + a private `getJson`/`postJson`.
- `src/views/{Setup,Login,WorldSelect}.svelte` (+ `Login.test.ts`, `WorldSelect.test.ts`) — moved.

Shell (`src/client/shell/`) after this slice:
- `src/App.svelte` — boot split; renders `<Entry>` for pre-world, `<Table>` in-world.
- `src/lib/api.ts` — loses `getConfig`/`login`/`setup`/`createWorld`; keeps `getMe`,
  `listWorlds`, `listWorldMembers`, `getUiState`/`putUiState`, `logout`, asset CRUD, types.
- `src/lib/views/` — **removed** (moved to entry).
- `src/App.test.ts` — narrowed to shell boot responsibilities, Entry mocked.

`listWorlds` and `getMe` are intentionally present in **both** the shell api and
`entryApi`: the shell needs `listWorlds` to validate the saved last-world and
`getMe` for identity/boot; entry needs `listWorlds` for world-select and `getMe`
as its auth probe. Each owns its copy (co-location decision, spec §4.2).

```ts
import type { ServerConfig, WorldEntry } from "@shadowcat/types";

async function getJson<T>(url: string): Promise<T> {
  const res = await fetch(url, { headers: { accept: "application/json" } });
  if (!res.ok) throw new Error(`${url} → ${res.status}`);
  return (await res.json()) as T;
}

async function postJson(url: string, body: unknown): Promise<Response> {
  return fetch(url, {
    method: "POST",
    headers: { "content-type": "application/json" },
    body: JSON.stringify(body),
  });
}

export function getConfig(): Promise<ServerConfig> {
  return getJson<ServerConfig>("/api/config");
}

/** Auth probe: the authenticated user's id, or null when unauthenticated (401). */
export async function getMe(): Promise<{ id: string } | null> {
  const res = await fetch("/api/me", { headers: { accept: "application/json" } });
  if (res.status === 401) return null;
  if (!res.ok) throw new Error(`/api/me → ${res.status}`);
  return (await res.json()) as { id: string };
}

export async function login(username: string, password: string): Promise<boolean> {
  const res = await postJson("/api/login", { username, password });
  return res.ok;
}

export async function setup(
  username: string,
  password: string,
  token?: string,
): Promise<{ ok: boolean; status: number }> {
  const body: Record<string, string> = { username, password };
  if (token) body.token = token;
  const res = await postJson("/api/setup", body);
  return { ok: res.ok, status: res.status };
}

export function listWorlds(): Promise<WorldEntry[]> {
  return getJson<WorldEntry[]>("/api/worlds");
}

export async function createWorld(name: string): Promise<WorldEntry> {
  const res = await postJson("/api/worlds", { name });
  if (!res.ok) throw new Error(`/api/worlds → ${res.status}`);
  return (await res.json()) as WorldEntry;
}
```

```svelte
<script lang="ts">
  import { getConfig, getMe } from "./entryApi";
  import Setup from "./views/Setup.svelte";
  import Login from "./views/Login.svelte";
  import WorldSelect from "./views/WorldSelect.svelte";

  // The pre-world flow is a self-contained package: the shell hands us callbacks
  // and we own the setup -> login -> world-select progression. Step is local state
  // (not the shell route), so a replacement entry package is a drop-in swap.
  let { onAuthenticated, onEnterWorld }: {
    onAuthenticated: () => void | Promise<void>;
    onEnterWorld: (worldId: string) => void;
  } = $props();

  type Step = "loading" | "setup" | "login" | "worlds";
  let step = $state<Step>("loading");

  async function decideStart() {
    try {
      const cfg = await getConfig();
      if (!cfg.initialized) { step = "setup"; return; }
      const me = await getMe();
      step = me ? "worlds" : "login";
    } catch {
      // A transient backend failure must not wedge entry on "loading"; fall to login.
      step = "login";
    }
  }
  decideStart();

  async function afterLogin() {
    // Let the shell fetch identity + apply saved session state (locale) before
    // world-select renders, mirroring the pre-split boot order.
    await onAuthenticated();
    step = "worlds";
  }
</script>

{#if step === "setup"}
  <Setup onDone={() => (step = "login")} />
{:else if step === "login"}
  <Login onAuthed={afterLogin} />
{:else if step === "worlds"}
  <WorldSelect onEnter={onEnterWorld} />
{:else}
  <p class="connecting">Loading…</p>
{/if}
```

```svelte
<script lang="ts">
  import { webSocketConnect } from "@shadowcat/core";
  import { Entry } from "@shadowcat/module-entry";
  import { getMe, listWorlds, type Me } from "./lib/api";
  import { loadSessionState, setLastWorld, flushOnUnload } from "./lib/sessionState.svelte";
  import { currentRoute, navigate } from "./lib/route.svelte";
  import { coreUi } from "./modules/core-ui/index";
  import { sceneTools } from "./modules/scene-tools/index";
  import { WorldSession } from "./lib/worldSession.svelte";
  import Table from "./lib/Table.svelte";

  let me = $state<Me | null>(null);
  let booted = $state(false);
  let session = $state<WorldSession | null>(null);

  async function boot() {
    try {
      me = await getMe();
      if (me) {
        const ui = await loadSessionState(); // applies the saved locale
        const last = ui.global.lastWorld;
        if (last) {
          // A transient /api/worlds failure degrades to entry, not a hard error.
          try {
            const worlds = await listWorlds();
            if (worlds.some((w) => w.id === last)) {
              enterWorld(last); // reload returns you to your last world
              return;
            }
            setLastWorld(null); // stale (deleted / revoked) — clear it
          } catch {
            // fall through to entry
          }
        }
      }
      navigate({ name: me ? "worlds" : "login" }); // pre-world; <Entry> picks the step
    } catch {
      navigate({ name: "login" });
    } finally {
      booted = true;
    }
  }
  boot();

  if (typeof window !== "undefined") {
    window.addEventListener("pagehide", flushOnUnload);
    window.addEventListener("visibilitychange", () => {
      if (document.visibilityState === "hidden") flushOnUnload();
    });
  }

  // Entry authenticated the user; fetch identity + apply saved session state
  // (locale) before entry advances to world-select — the pre-split boot order.
  async function onAuthenticated() {
    try {
      me = await getMe();
      await loadSessionState();
    } catch {
      me = null;
    }
  }

  function enterWorld(worldId: string) {
    if (!me) return;
    const wsUrl =
      (location.protocol === "https:" ? "wss:" : "ws:") +
      "//" + location.host + "/ws?world=" + worldId;
    const s = new WorldSession({ selfId: me.id, connect: webSocketConnect(wsUrl), coreUiModule: coreUi, featureModules: [sceneTools] });
    session = s;
    void s.enter(worldId);
    setLastWorld(worldId);
    navigate({ name: "world", id: worldId });
  }

  function leaveWorld() {
    session?.leave();
    session = null;
    setLastWorld(null);
    navigate({ name: "worlds" });
  }

  const route = $derived(currentRoute());
</script>

{#if !booted}
  <p class="connecting">Loading…</p>
{:else if route.name === "world" && session?.role && session?.world}
  <Table {session} {leaveWorld} />
{:else if route.name === "world"}
  <p class="connecting">Connecting…</p>
{:else}
  <Entry {onAuthenticated} onEnterWorld={enterWorld} />
{/if}
```

(Note: `WorldSession`'s `{ coreUiModule, featureModules }` API is unchanged in this
